## Project shape (unchanged fundamentals)

- **Binary crate** (`trellis`), no lib target. Extra bins in `src/bin/` include
  the model via `#[path = "../model.rs"] mod model;`.
- Source: `src/main.rs`, `src/app.rs` (~1100 loc, eframe App + menus + persistence),
  `src/model.rs` (data model + all document ops + exports/imports),
  `src/canvas.rs` (basket canvas: cards, toolbar, interactions),
  `src/tree.rs`, `src/api.rs` (HTTP agent API), `src/images.rs` (texture cache).
- Data model: `Document { nodes: HashMap<NodeId,Node>, roots, next_node_id, next_card_id, next_group_id }`.
  `Node { id, title, parent, children, cards: Vec<Card>, groups: Vec<CardGroup>, expanded, color }`.
  `Card { id, pos, size, title, body, color, kind, group, docked_to, editing(skip) }`.
  `CardKind = Text | Code{lang} | Checklist{items} | Image{data,name}`.
- Persistence: RON. `save()` explicit (Ctrl+S / menu) **and** `on_exit()`
  best-effort autosave to the working file or `autosave.ron`. **SIGKILL bypasses
  `on_exit`** — used during testing to discard edits to the loaded doc.
- Rendering: markdown via **vendored** `egui_commonmark` (see 0.15.0). Fenced code
  uses syntect highlighting.

## Feature details

### 0.8.1 — Primary-selection in singleline fields
Bug: card **title** (and code `lang`, checklist item) were plain
`ui.add(TextEdit::singleline)` with none of the X11 primary-selection wiring the
body editor had, so middle-click paste there did nothing.

- `singleline_primary(ui, id, initial, build) -> (String, bool, Response)` in
  `canvas.rs`: mirrors selection to PRIMARY and pastes PRIMARY at the cursor on
  middle-click. Routed title / lang / checklist-item through it.
- Backing helpers: `mirror_selection_to_primary`, `take_primary_selection`,
  `set_primary_selection` — shell out to **`xclip`/`xsel`** (arboard/egui can't
  serve PRIMARY). **Linux/X11 only**; needs xclip or xsel installed.

### 0.9.0 — Text color picker
- Toolbar (Text card edit): `color_edit_button_srgb(&mut rgb)` (persisted in egui
  memory `Id::new("trellis_text_color")`) + a red **"A"** apply button.
- `wrap_color(text, sel, rgb)` wraps selection in
  `<span style="color:#rrggbb">…</span>`, keeping whitespace outside the span.
- Colors in **HTML/PDF export** immediately; **live** rendering added in 0.15.0.

### 0.10.0 — Image titles + line breaks
- `title_field(ui, card, actions)` in `canvas.rs` — shared title editor (uses
  `singleline_primary`), used by **Text and Image** cards.
- `supports_edit()` now includes `Image` → image cards get the edit/view toggle
  and a title field (title bar shows the name to tell images apart).
- `hard_wrap(md)` in `model.rs`: converts single `\n` to a Markdown hard break
  (two trailing spaces), **skipping fenced code blocks**. Applied in `md_to_html`
  (export) and the Text card viewer. CommonMark treats lone newlines as soft
  breaks (a space); this makes each Enter a visible line break.

### 0.11.0 — Grouping + docking
**Model (`model.rs`):**
- `type GroupId = u64`; `struct CardGroup { id, title, color: [u8;3] }`.
- `Card.group: Option<GroupId>`, `Card.docked_to: Option<CardId>` — both `#[serde(default)]`.
- `Node.groups: Vec<CardGroup>` (`#[serde(default)]`); `Document.next_group_id`
  (`#[serde(default = "default_next_id")]`) — **existing RON docs load unchanged**.
- Methods: `group_cards(node, &[CardId], title) -> Option<GroupId>` (needs ≥2 existing
  cards), `ungroup`, `set_group_title`, `set_group_color`, `move_group`,
  `dock_card(node, child, anchor)` (**cycle-safe** — refuses if anchor is in child's
  subtree), `detach_card`, `move_card_tree(node, card, delta)` (moves card + dock
  subtree), private `dock_tree_ids`, free fn `prune_groups(node)`.
- `remove_card` now detaches dependents and prunes empty groups.

**Canvas (`canvas.rs`):**
- New `CanvasAction`: `ToggleSelect, ClearSelection, GroupSelected, Ungroup,
  MoveGroup, SetGroupTitle, SetGroupColor, DockCard, DetachCard`.
- `canvas::ui(...)` gained `selection: &HashSet<CardId>`.
- **Selection**: Ctrl/Cmd+click a title bar toggles selection (amber outline);
  plain click clears; empty-canvas click clears. Floating **"Group N cards"**
  button (top-left) appears when ≥2 selected.
- **Group container**: bbox of members drawn behind cards + a **header** tab;
  drag header → `MoveGroup`; right-click header → `group_menu` (rename/color/ungroup).
- **Docking**: `dragging` detected via `ctx.is_being_dragged(card_handle_id)`;
  memory key `"canvas_dragging_card"`. On drag start → `DetachCard`; on drop →
  `DockCard` to the top card under the pointer (excluding the dragged subtree);
  **green highlight** on the hover target; faint **connector lines**; **dot** on a
  docked card's title bar. All docking gated by **Dock mode** (0.11 added the
  toggle; see 0.12 note — the `dock_mode` bool + canvas "Dock" button + Settings).

**App (`app.rs`):** `card_sel: HashSet<CardId>`, `card_sel_node: Option<NodeId>`
(reset on node change). `MoveCard` now calls `move_card_tree`. Handles all new
actions. `dock_mode` persisted (key `"dock_mode"`).

### 0.12.0 — Snap mode
- `app.rs`: `snap_mode: bool` (persisted key `"snap_mode"`), Settings checkbox,
  canvas **"Snap"** toggle button (next to "Dock"), `ToggleSnapMode` action.
- `canvas.rs`: `const SNAP_DIST = 8.0`. `snap_position(pos, size, others, self_id,
  threshold) -> (Pos2, Option<f32> guide_x, Option<f32> guide_y)` snaps each axis
  independently to the nearest other-card edge. Drag uses a **grab-offset** stored
  in memory (`("card_grab", id)`) so snapping tracks the pointer without drift.
  Amber **guide lines** drawn at snapped edges.

### 0.13.0 — PDF/PNG/GIF export + full API
**Deps added:** `printpdf = "0.7"`, `ab_glyph = "0.2"`.

**Export (`model.rs`):**
- `EXPORT_FONT = include_bytes!("../assets/DejaVuSans.ttf")`.
- `struct ExportLine { text, size }`, `export_lines()` / `export_node_lines()`
  flatten the whole tree to sized lines.
- `export_pdf() -> Result<Vec<u8>, String>` — printpdf, A4, **embeds DejaVuSans**,
  paginated. (printpdf 0.7 uses `Mm(f32)` and `use_text(text, f32, ..)`.)
- `export_image(gif: bool) -> Result<Vec<u8>, String>` — rasterize lines with
  `ab_glyph` onto an `image::RgbaImage` (one tall page), encode PNG or GIF via the
  `image` crate. Whole-document; **large docs → very tall PNG (slow)**, PDF paginates.
- Helpers: `text_width`, `wrap_text` (greedy word wrap using ab_glyph advances),
  `draw_text` (per-glyph outline rasterize, black-on-white).
- `app.rs`: `export_pdf()`, `export_image(gif)` (rfd save dialog); menu items under
  **File → Export**: PDF…, PNG image…, GIF image….

**API (`api.rs`) — see API.md for the full surface:**
- Card `PATCH` now accepts `color, lang, pos, size, items` (was title/body only)
  and **returns the updated card**. Card JSON now reports `pos, size, color,
  group, docked_to`. Node JSON now includes `groups`.
- **Groups:** `GET/POST/PATCH/DELETE /api/nodes/{id}/groups[/{gid}]`.
- **Docking:** `POST/DELETE /api/nodes/{id}/cards/{cid}/dock` (body `{anchor}`).
- **Export:** `GET /api/export?format=markdown|html|json|pdf|png|gif` — text inline
  (`content`), binary base64 (`base64`).
- New DTOs: `UpdateCardInput` (now `pub`, expanded), `CreateGroupInput`,
  `UpdateGroupInput`, `DockInput`. Helpers: `groups_json`, `group_exists`,
  `export_response`. `base64::Engine` imported for export encoding.

### 0.14.0 — Auto-continue lists
- In the Text body editor, **before** `TextEdit::show`, a plain Enter (no modifiers)
  in the focused editor is intercepted: read the cursor line, `list_enter(line)`
  decides `Continue(String)` (insert `\n` + next marker) or `Exit` (clear an empty
  marker). The Enter `Event::Key` is removed via `ui.input_mut(|i| i.events.retain(..))`
  so the editor doesn't also insert a newline. The result feeds the existing
  `edited` path (SetBody + cursor + focus).
- Handles numbered (`1.`, `1)`), bullets (`-`,`*`,`+`), tasks (`- [ ]`), preserving
  indent. Empty item exits the list. **Shift+Enter** = plain newline.
- Helpers: `line_start(text, at)`, `enum ListEnter`, `list_enter(line)`.

### 0.14.1 — Numbered button increments
- `numbered_prefix(text, sel)` numbers each touched line `1. `, `2. `, `3. …`
  (the numbered toolbar button previously used `line_prefix(.., "1. ")`, giving
  every line `1.`). Correctly tracks cursor offsets incl. wider markers (`10. `).

### 0.15.0 — Live inline text color (vendored renderer)
Upstream `egui_commonmark` **drops all inline HTML** (`Event::InlineHtml(_) => {}`),
so color spans never showed live. Fix = vendor + patch, keeping full fidelity.

- `vendor/egui_commonmark/` and `vendor/egui_commonmark_backend/` — copied from
  crates.io 0.18.0 (MIT/Apache, LICENSE-MIT kept). Optional `egui_commonmark_macros`
  dep + `macros`/`nightly`/`dump-macro` features **dropped** (all `#[cfg]`-gated).
- `Cargo.toml`: `egui_commonmark = { path = "vendor/egui_commonmark", features = ["better_syntax_highlighting"] }`.
- **Backend patch** (`vendor/egui_commonmark_backend/src/misc.rs`): `Style` gained
  `color: Option<egui::Color32>`, applied at the end of `to_richtext` via `.color()`.
- **Frontend patch** (`vendor/egui_commonmark/src/parsers/pulldown.rs`): the
  `InlineHtml` arm parses `<span style="color:#rrggbb">` → set `text_style.color`,
  `</span>` → clear it. New `parse_span_color(tag) -> Option<Color32>`.
- Result: color shows **live** in the card view AND in export; all other markdown
  (bold, tables, syntax-highlighted code, lists) unchanged. Single `Option`, so
  nested color spans don't stack (fine for the app's usage).

## How the GUI was verified (repeatable recipe)

No formal e2e harness — driven live with **xdotool** on `DISPLAY=:0`, screenshots
via ImageMagick `import`:

```sh
# build + launch detached (survives shell signals)
cargo build --bin trellis
setsid ./target/debug/trellis >/tmp/log 2>&1 </dev/null &
PID=$(ps -C trellis -o pid= | tr -d ' ' | head -1)
WID=$(xdotool search --pid $PID | tail -1)
xdotool windowactivate $WID
eval $(xdotool getwindowgeometry --shell $WID)   # sets X,Y,WIDTH,HEIGHT
xdotool mousemove $((X+dx)) $((Y+dy)) click 1     # screen = window origin + in-window offset
import -window $WID /tmp/shot.png                 # or: import -window $WID -crop WxH+x+y +repage
killall -9 trellis                                # SIGKILL: skips on_exit autosave → discards test edits
```

- Middle-click paste round-trip verified with `xclip -selection primary`.
- PDF eyeballed by rendering with `pdftoppm -png`.
- **Data safety:** all test instances loaded the real `Notes.ron`; killing with
  `killall -9` (SIGKILL) skips `on_exit`, so edits were never saved. Confirmed
  `Notes.ron` mtime stayed unchanged each time.

## Release process (followed every version)

1. Bump `version` in `Cargo.toml`. 2. `cargo test --bin trellis`. 3. `cargo build
--release` (LTO, ~1 min; incremental debug ~3 s). 4. Commit. 5. Annotated tag:
`git tag -a vX.Y.Z -m "Trellis vX.Y.Z"` + push. 6. `gh release create vX.Y.Z
<target/release/trellis copied to trellis-X.Y.Z-linux-x86_64> --title "Trellis
vX.Y.Z" --notes "..."`.

**Rule: no self-attribution anywhere** (commits, tags, release notes). Verified
each release with a grep for claude/anthropic/co-author/"generated with".

## Gotchas / constraints for next time

- **Vendored markdown renderer** now lives in `vendor/` — edit there, not the
  crates.io copy. Re-derive from upstream 0.18.0 if bumping egui.
- **PRIMARY selection** (middle-click paste, color-out) needs `xclip`/`xsel`; X11
  only. Wayland unsupported for PRIMARY.
- **Image/PDF export is whole-document.** Huge docs make a giant, slow PNG; prefer
  PDF (paginates). Possible future work: per-basket image export or capped height.
- **Grouping is not a mode** (unlike Dock/Snap toggles) — it's select-then-Group.
- Known rough edge: a group **header tab can tuck under a card directly above** the
  container. Candidate fix: draw the header inside the top of the box.
- `cargo test` has **no lib target**; always `--bin trellis`.
- Recent version bumps were tagged/released this session; earlier gap (v0.9–v0.12
  were bumped but originally untagged) has been closed.
