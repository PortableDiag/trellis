## Feature details

### 0.16.0 — Copy button on card title bars
- New `CanvasAction`s and title-bar button (left of the edit/view toggle) that
  copies a card's text to **both** the system clipboard (`ctx.copy_text`) and
  the X11 PRIMARY selection (`set_primary_selection`, via xclip/xsel).
- `copyable_text(card) -> Option<String>` in `canvas.rs`: Text/Code → body;
  Checklist → Markdown task lines (`- [x] …`); **Table → CSV** (added 0.18.0);
  Image → `None` (no button).
- `copy_both(ui, text)` clears the PRIMARY dedup key first so re-copying the
  same text after another app overwrote PRIMARY still works.
- The docked-card indicator dot was nudged left to make room for the button.

### 0.16.1 / 0.16.2 — Dialog parenting (+ crash fix)
- **Problem:** `rfd` file/message dialogs had no transient-for parent, so the
  WM could stack them *behind* the Trellis window.
- **Fix:** `DialogParent` struct wraps raw window+display handles captured each
  frame from `eframe::Frame` (via `raw-window-handle` 0.6, added to Cargo.toml).
  All 13 dialog call sites go through `self.file_dialog()` / `self.message_dialog()`
  helpers that call `.set_parent(p)`.
- **0.16.2 crash:** the initial helper bodies accidentally called *themselves*
  (`let d = self.file_dialog();`) → infinite recursion → stack overflow on any
  dialog open. Fixed to `rfd::FileDialog::new()`. **Lesson: the compiler's
  `unconditional_recursion` warning was present but filtered out of build
  output — don't grep-filter away warnings.**

### 0.17.0 — Multi-image cards + lightbox
**Model (`model.rs`):**
- `struct ImageEntry { data: Vec<u8>, name: String }`.
- `CardKind::Image { data, name, extra: Vec<ImageEntry> }` — `extra` is
  `#[serde(default)]`, so **single-image docs load unchanged** (first image
  stays in `data`/`name`; extras append to `extra`).
- `CardKind::images() -> Vec<(&[u8], &str)>` — primary (if non-empty) then extra,
  in display order. Used everywhere images are enumerated.
- `Document::add_image(node, card, bytes, name)` (first load fills primary slot),
  `remove_image(node, card, idx)` (removing primary promotes next extra).
- Export/search sites updated to iterate `images()`.

**Texture cache (`images.rs`):** keyed by `(CardId, usize)` (card + image index),
`forget(card)` drops all of a card's textures.

**Canvas (`canvas.rs`):**
- Grid layout: `grid_cols(n)` → 1 image full-width, 2–4 in two columns, 5+ three.
- Each image is clickable; **double-click → `OpenLightbox(card, idx)`**;
  right-click → View / Remove image. Footer shows "N images" + "add image".

**Lightbox (`app.rs`):** `struct Lightbox { node, card, index, zoom, pan }`,
`lightbox_ui(ctx)` drawn as a foreground `Area` over a dark backdrop:
- scroll or `+`/`-` to zoom, drag to pan, double-click toggles fit ↔ 2×,
  `←`/`→` (keys **and** on-screen ◀ ▶ buttons) cycle images, Esc / `×` /
  backdrop click closes. Caption shows `name — i/n · zoom% · hints`.

### 0.17.1 — Right-click card placement
- **Bug:** the right-click "Add card" menu re-read `interact_pointer_pos()` on
  the frame a menu item was clicked — by then the pointer is on the menu, so it
  was `None` → fell back to hardcoded world `(40,40)` ("the top area").
- **Fix:** capture the click's world position into egui temp memory
  (`menu_world_key`) on `secondary_clicked()`; the menu items read that. Fallback
  is now the **center of the visible canvas**, not the origin. Applies to all
  card kinds and Paste. (Double-click text-card placement was already correct.)

### 0.18.0 — Table / spreadsheet cards
**Model (`model.rs`):**
- `struct TableCell { text: String, bg: Option<[u8;3]>, fg: Option<[u8;3]> }`
  (both colors `#[serde(default)]`).
- `struct TableData { rows: Vec<Vec<TableCell>>, col_widths: Vec<f32>, header: bool }`
  (`col_widths`/`header` defaulted; `header` defaults true).
  Methods: `empty(rows,cols)`, `n_cols()`, `col_width(c)`, `from_values(Vec<Vec<String>>)`,
  `to_csv()`, `to_xlsx() -> Result<Vec<u8>,String>` (colors + bold header preserved).
- `const TABLE_DEFAULT_COL_W: f32 = 110.0`.
- Free fns: `csv_to_values(&[u8])`, `xlsx_to_values(&[u8])` (first sheet, via calamine).
- `CardKind::Table { table: TableData }`.
- Document ops (all return bool, keep grid rectangular, never < 1 row/col):
  `table_set_cell`, `table_set_bg`, `table_set_fg`, `table_insert_row`,
  `table_remove_row`, `table_insert_col`, `table_remove_col`,
  `table_set_col_width`, `table_toggle_header`, `table_replace(values)`.
  Private `table_mut(node, card)` helper.
- Exports: HTML → real `<table>` with inline `background:`/`color:` styles and
  `<th>` for the header row; Markdown → pipe table with `| --- |` separator;
  PDF/image → one `a | b | c` line per row; search → all cell text.

**Canvas (`canvas.rs`):**
- `table_ui(ui, card, table, actions)` inside a horizontal `ScrollArea`.
  Edit mode: toolbar (`+ row`, `+ col`, `header` checkbox, `Import…`, `CSV…`,
  `XLSX…`), color row (bg picker + `fill`, fg picker + `A`, `clear`, shows the
  focused cell like `cell B2`), column-letter header strip with per-column
  context menu (insert left/right, delete) and drag-resize grips, then the grid
  of `TextEdit::singleline` cells. View mode: read-only galley-rendered grid
  with cell bg/fg colors and bold/shaded header.
- Focused cell tracked in egui temp memory (`table_focus` key) so the color
  buttons know which cell to apply to. Color pickers persist their last value in
  memory (`trellis_table_bg` / `trellis_table_fg`).
- `col_letter(c)` → A, B, …, Z, AA, AB…. New `CanvasAction`s: `TableSetCell`,
  `TableSetBg`, `TableSetFg`, `TableInsertRow`, `TableRemoveRow`,
  `TableInsertCol`, `TableRemoveCol`, `TableSetColWidth`, `TableToggleHeader`,
  `TableImport`, `TableExportCsv`, `TableExportXlsx`.
- Right-click "Add card" menu gained **Table** (creates `TableData::empty(3,3)`).
- `supports_edit()` now includes `Table`; `Card::new` starts tables in edit mode.

**App (`app.rs`):** handles all `Table*` actions (each sets `dirty`).
`table_import(node, card)` (CSV or XLSX by extension → `table_replace`),
`table_export(node, card, xlsx: bool)` (rfd save dialog; CSV bytes or `to_xlsx()`).

**API (`api.rs`):** `kind: "table"` creates a 3×3 table. Card JSON for tables
adds `header` and `rows` (each cell `{text, bg, fg}`). `UpdateCardInput` gained
`rows: Option<Vec<Vec<String>>>` → PATCH replaces all cell values (colors reset).

**Deps added (Cargo.toml):** `csv = "1"`, `calamine = "0.26"` (xlsx read),
`rust_xlsxwriter = "0.79"` (xlsx write).

## Agent API surface (localhost, key-gated — see API.md)

Unchanged endpoints from prior sessions plus table support:
- Card `kind` is now one of `text | code | checklist | table | image`.
- `POST /api/nodes/{id}/cards` accepts `kind: "table"` (→ 3×3 empty table).
- Card JSON for a table card includes `header` (bool) and `rows`
  (`[[{text,bg,fg}, …], …]`).
- `PATCH /api/nodes/{id}/cards/{cid}` accepts `rows: [["a","b"],…]` to replace
  all cell values (colors reset). Existing fields: `title, body, color, lang,
  pos, size, items`.
- Image cards report `image_name`, `image_names`, `bytes` (from 0.17.0).

**Reading the API key/port from disk:** they live in Trellis's eframe storage
`~/.local/share/trellis/app.ron` (a RON map of string→string), keys `api_key`
and `api_port` (default port 7373). Meet Scribe reads them from there.

## GUI verification recipe (updated — important boundaries)

Driven live with **xdotool** on `DISPLAY=:0`, screenshots via ImageMagick
`import`. **HARD-LEARNED CONSTRAINTS (the operator was actively using the
machine this session):**
- **Never `killall trellis`** — the operator runs their real notes session
  (loads `/media/veracrypt2/Notes.ron`). Kill only the *exact PID you spawned*.
- **Never `windowmove` a test window to another monitor** and never leave a hung
  `import` (always pass a real WINDOW id, never a PID; wrap in `timeout`).
- Run test instances under a **fake `HOME`** pointing at a scratch fixture doc,
  so the real notes file is never opened or written:
  ```sh
  S=<scratchpad>; mkdir -p $S/home/.local/share/trellis
  printf '{\n"theme":"Dark",\n"last_doc_path":"%s/fixture.ron",\n}' "$S" > $S/home/.local/share/trellis/app.ron
  HOME=$S/home setsid ./target/debug/trellis >/dev/null 2>&1 </dev/null &
  # identify our PID by its HOME env:
  for p in $(pgrep -x trellis); do tr '\0' '\n' </proc/$p/environ | grep -q "$S/home" && MYPID=$p; done
  MYWID=$(DISPLAY=:0 xdotool search --pid $MYPID | tail -1)
  timeout 10 env DISPLAY=:0 import -window $MYWID $S/shot.png
  kill -9 $MYPID   # exact PID only
  ```
- `cargo test` does **not** rebuild the run binary — `cargo build --bin trellis`
  before launching, or the GUI runs stale code.
- Table card + fixture verified this way (view mode + edit toolbar both shot).

1. Bump `version` in Cargo.toml. 2. `cargo test --bin trellis`. 3.
`cargo build --release`. 4. Commit. 5. `git tag -a vX.Y.Z -m "Trellis vX.Y.Z"` +
push. 6. `gh release create vX.Y.Z <binary copied to
trellis-X.Y.Z-linux-x86_64> --title … --notes …`.

**Rule: no self-attribution anywhere** (commits, tags, releases, code, docs) —
no assistant name, co-author trailer, or "generated with" line. Verified each

# Meet Scribe — new sibling project

**Location:** `/media/veracrypt1/Rust/meet-scribe`
**GitHub:** `github.com/PortableDiag/meet-scribe` (**private**), releases
v0.1.0 → v0.1.3.

**What it is:** a one-button, fully **local** Linux meeting recorder. Captures
what you hear (USB-headset monitor = "Them") and what you say (mic = "Me") via
PipeWire/ffmpeg, transcribes on-device with faster-whisper on the GPU, and
**streams the transcript live into a Trellis `Meetings → <date> Meeting` node**
via the agent API so you can watch it working mid-call. Nothing leaves the
machine (explicit operator requirement).

**Rust + Python split:** Rust binary is the GUI/orchestrator (never blocks the
UI thread on subprocess/HTTP); a persistent Python worker
(`transcriber.py`) holds the faster-whisper model and serves JSON-lines
requests over stdin/stdout.

### Files
- `src/main.rs` — egui GUI: one big Record/Stop button, status, three status
  dots (USB headset / Trellis API / transcriber installed), auto-record +
  live-stream toggles, Install button, Open-recordings / View-log buttons.
  Single-instance lockfile (`app.pid`), orphan-process reaping on launch,
  bundled DejaVu font (status glyphs), embedded window icon, close-intercept
  (finish transcript before exit). Background thread polls devices + Trellis
  health every 3 s (off the UI thread — this was the v0.1.0 freeze bug).
- `src/audio.rs` — PipeWire device discovery via `pw-dump` JSON.
  `Devices { sink, source, label, usb }`. `pick_devices()` prefers a USB
  headset (both directions on a USB card), falls back to first analog.
  `usb_headset()` for the auto-record watcher.
- `src/capture.rs` — two `ffmpeg` children writing **20 s WAV segment chunks**
  (16 kHz mono) from `<sink>.monitor` and the mic. `Recorder::start/stop/alive`,
  `chunks(dir, tag)`, `concat(dir, tag)` (lossless concat for the final pass).
  Chunked so transcription can stream during the meeting.
- `src/transcribe.rs` — bridge to the Python worker. `Worker::spawn(model)`
  (blocks until model loaded), `text(path)` (live plain-text chunk),
  `segments(path)` (final timestamped pass). `install_env()` builds the venv +
  pip installs. CUDA libs come from **pip wheels** (`nvidia-cublas-cu12`,
  `nvidia-cudnn-cu12`, `nvidia-cuda-runtime-cu12`) — **no system CUDA needed**;
  `LD_LIBRARY_PATH` is assembled to point at them.
- `src/session.rs` — the meeting session thread: record → stream chunk
  transcripts into a live Trellis card → on stop, finalize chunks, run a
  full-quality timestamped pass, write `transcript.md`, update the card.
  `Shared` struct is the GUI↔session channel. Raw WAVs hit disk first so audio
  is never lost if transcription/Trellis fail.
- `src/trellis.rs` — minimal agent-API client. `from_trellis_settings()` reads
  key/port from `~/.local/share/trellis/app.ron`. `ensure_root("Meetings")`,
  `create_node`, `create_text_card`, `set_card_body/title`, `resize_card`,
  `health`.
- `transcriber.py` — persistent faster-whisper worker (`large-v3-turbo`).
  Tries CUDA (float16) with a forced warm-up on 1 s of silence to surface GPU
  errors at load, falls back to CPU (int8). JSON-lines protocol:
  request `{path, tag, id, detail}` → response `{text}` or `{segments:[{s,e,text}]}`.
- `assets/` — `icon.png`, `meet-scribe.desktop`, bundled `DejaVuSans.ttf`.
- `README.md`, `.gitignore`.

### Runtime layout (on the machine, not in the repo)
- `~/.local/share/meet-scribe/venv` — Python venv (faster-whisper + CUDA wheels).
- `~/.local/share/meet-scribe/transcriber.py` — worker script (rewritten from
  the embedded copy on each launch).
- `~/.local/share/meet-scribe/sessions/<timestamp>/` — WAV chunks + `transcript.md`.
- `~/.local/share/meet-scribe/meet-scribe.log` — debug log ("View log" button).
- `~/.local/share/meet-scribe/app.pid` — single-instance lock.
- Model in the Hugging Face cache (`~/.cache/huggingface`, ~1.6 GB).
- Installed binary: `~/.local/bin/meet-scribe`; launcher in
  `~/.local/share/applications/`.
